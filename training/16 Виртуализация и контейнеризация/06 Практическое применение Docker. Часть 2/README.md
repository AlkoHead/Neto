![task_006_0](img/task_006_00.JPG)

В `dive` мы можем послойно рассмотреть образ `docker` и увидеть в каком слое добавляется интересующий нас файл. В `Layer Detail` нас интересует пункт `Digest`.

Последовательность использования `docker save`
```bash
# Сохранить образ в tar архив
docker save hashicorp/terraform:latest -o terraform-image.tar

# Распакуем весь архив во временную директорию
mkdir -p terraform-extract
tar -xf terraform-image.tar -C terraform-extract/

# сделал ещё одну папку куда распоковал по пункт `Digest`
mkdir -p on_terr 
tar -xf terraform-extract/blobs/sha256/56cfe20bb05ee957a1f2490531f6972e754fa00a625a3ad467f317f5fff67e72 -C on_terr/

# просмотр
ls on_terr/bin/
```

```bash

docker create --name temp-terraform hashicorp/terraform:latest
# история образа
docker history hashicorp/terraform:latest
# копирование файла из контейнера
docker cp temp-terraform:/bin/terraform ./terraform
# удаление временного контейнера
docker rm temp-terraform
#  проверка скопированного файла
ls -la terraform
```
